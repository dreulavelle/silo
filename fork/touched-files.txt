# Upstream files this fork modifies.
#
# Every entry here is a rebase liability. The whole strategy of this fork is to
# keep this list SHORT and to keep each edit SMALL and STRUCTURALLY STABLE
# (top-of-function guards, map literal entries, single call sites) so that
# upstream churn inside those files rarely collides with us.
#
# scripts/fork-sync.sh cross-references this list against the files upstream has
# touched since fork/upstream.pin and warns before a rebase is attempted.
#
# Anything NOT in this list must live in a new file under a fork-owned path:
#   internal/strm/...          fork-owned Go package
#   internal/scanpush/...      fork-owned Go package
#   migrations/sql/9*.sql      fork-owned migrations (sort last, see FORK.md)
#   fork/, scripts/fork-*      fork tooling
#
# Format: one repo-relative path per line. '#' comments and blanks ignored.

internal/scanner/scanner.go
internal/playback/directplay.go

# Transcoding has to resolve a placeholder before ffmpeg opens it: ffmpeg reads
# the input itself and a .strm is not a container it understands. The edit is
# one helper plus one call at each of the two ffmpeg launch sites, so it stays
# narrow — but this is the largest upstream file the fork touches, so churn here
# is the most likely source of a rebase conflict.
internal/playback/transcode.go

# Missing playback metadata for a placeholder is repaired by resolving it and
# probing the stream behind it, rather than ffprobing a file that holds a URL.
internal/scanner/probe_repair.go

# A placeholder whose metadata could not be resolved plans a conservative
# transcode instead of refusing to play.
internal/playback/plan_v3.go

# One block starting the fork-owned scanpush consumer.
cmd/silo/main.go

# New test files in upstream packages. These carry less rebase risk than a
# modified file — nothing to conflict with unless upstream adds a file of the
# same name — but they cannot live under a fork-owned path, because they test
# unexported functions in the package they sit in.
internal/scanner/strm_probe_test.go
internal/playback/strm_plan_test.go

# Default image points at this fork's published image rather than upstream's.
# Both are one-line value swaps on an already-parameterized setting
# (${SILO_IMAGE:-...}), so they rebase cleanly unless upstream restructures the
# compose entirely.
docker-compose.yml
